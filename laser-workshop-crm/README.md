# 📚 Полная документация Laser Workshop CRM

## 🎯 Описание системы

CRM-система для управления лазерной мастерской с поддержкой двух станков:
- **JPT M7 60W** — оптоволоконный лазерный маркер (металлы)
- **Ortur LM3 10W** — диодный гравёр (неметаллы)

> ⚠️ **Важно**: Типы станков скрыты от клиентов и используются только для внутренней автоматизации!

## 🧮 11 Алгоритмов калькулятора

### 2.1 fixed — Штучный товар
**Применение**: Фляжки, жетоны, брелоки  
**Формула**: `Цена * Количество`

### 2.2 area_cm2 — Площадь см²
**Применение**: Шильды, гравировка дерева/кожи  
**Формула**: `(Длина/10 * Ширина/10) * Цена_за_см² * Количество`

### 2.3 meter_thickness — Резка по толщине
**Применение**: Резка фанеры Ortur  
**Формула**: `Метры_реза * (Цена * (Толщина_мм / 3.0))`

### 2.4 per_minute — Поминутная оплата
**Применение**: Долгая гравировка JPT  
**Формула**: `Минуты * Цена_за_минуту`

### 2.5 per_char — За символ
**Применение**: Кольца, ручки, гравировка текста  
**Формула**: `Кол-во_символов * Цена_за_символ * Количество`

### 2.6 vector_length — Длина вектора
**Применение**: Промышленная резка  
**Формула**: `Длина_вектора(м) * Цена_за_метр`

### 2.7 setup_batch — B2B тираж
**Применение**: Оптовые заказы  
**Формула**: `Настройка_станка + (Цена_шт * Тираж)`

### 2.8 photo_raster — Фото растр
**Применение**: Фото на дереве/металле  
**Формула**: `Площадь * Цена * Коэфф_DPI`

### 2.9 cylindrical — Цилиндрические объекты
**Применение**: Термосы, кружки  
**Формула**: `(Диаметр * π * Длина/100) * Цена`

### 2.10 volume_3d — 3D клише
**Применение**: Объемная гравировка MOPA  
**Формула**: `Площадь * Глубина(мм) * Цена`

### 2.11 material_and_cut — Материал + Резка
**Применение**: Комплексные заказы  
**Формула**: `Площадь_материала*Цена + Рез*Цена`

## 💰 Система скидок

### Оптовые скидки (автоматически)
| Количество | Скидка |
|-----------|--------|
| 10+ шт | 5% |
| 20+ шт | 10% |
| 50+ шт | 15% |
| 100+ шт | 20% |

### Промокоды
Создаются в админке, применяются вручную клиентом.

### Кэшбек
- **5%** баллами после завершения заказа (статус `done`)
- 1 балл = 1 рубль
- Можно оплачивать до 50% заказа кэшбеком

## 🌐 Web API

### Основные эндпоинты

#### Заказы
```
GET    /api/orders              # Список заказов
GET    /api/orders/<id>         # Заказ по ID
POST   /api/orders              # Создание заказа
PUT    /api/orders/<id>/status  # Обновление статуса
GET    /api/orders/<id>/history # История заказа
```

#### Клиенты
```
GET    /api/clients             # Список клиентов
GET    /api/clients/<id>        # Клиент по ID
```

#### Услуги
```
GET    /api/services            # Список услуг
```

#### Калькулятор
```
POST   /api/calculate           # Расчёт стоимости
```

#### Статистика
```
GET    /api/stats/dashboard     # Статистика дашборда
GET    /api/stats/revenue       # Выручка по дням
```

#### Экспорт/Бэкапы
```
GET    /api/export/orders       # Экспорт заказов CSV
POST   /api/backup/create       # Создать бэкап
GET    /api/backup/list         # Список бэкапов
GET    /api/backup/download/<f> # Скачать бэкап
```

## 🚀 Запуск системы

### 1. Установка зависимостей
```bash
pip install flask requests vk_api
```

### 2. Настройка переменных окружения
```bash
export VK_API_TOKEN=your_token
```

### 4. Запуск сервера
```bash
python backend/app.py
```

```
http://localhost:5000
```

## 🔄 Жизненный цикл заказа

```
NEW ➔ PROCESSING ➔ DONE ➔ DELIVERED
```

1. **NEW** — Заказ создан, ожидает подтверждения
2. **PROCESSING** — В работе у мастера
3. **DONE** — Готов, начислен кэшбек (5%)
4. **DELIVERED** — Выдан клиенту

При смене статуса → автоматическое уведомление в VK!
