## 🗄️ ЧАСТЬ 2: Настройка Samba для доступа с ПК

### 2.1 Создание пользователя Samba

```bash
# Добавление пользователя andor в Samba
sudo smbpasswd -a andor
# Введите пароль при запросе (например: sambapass123)

# Активация пользователя
sudo smbpasswd -e andor
```

### 2.2 Настройка конфигурации Samba

```bash
# Резервная копия конфига
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.backup

# Редактирование конфига
sudo nano /etc/samba/smb.conf
```

**Добавьте в КОНЕЦ файла:**

```ini
# ========================================
# Laser Workshop Project Share
# ========================================

[global]
   workgroup = WORKGROUP
   server string = Raspberry Pi Samba Server
   security = user
   map to guest = never
   dns proxy = no

# Общий доступ к проекту лазерной мастерской
[laser-workshop]
   comment = Laser Workshop CRM Phase 3
   path = /home/andor/laser-workshop-phase3
   browseable = yes
   read only = no
   guest ok = no
   valid users = andor
   create mask = 0755
   directory mask = 0755
   force user = andor
   force group = andor
```

**Сохранение и выход:** `Ctrl+O` → `Enter` → `Ctrl+X`

### 2.3 Перезапуск Samba

```bash
# Проверка конфигурации
sudo testparm

# Перезапуск сервисов
sudo systemctl restart smbd nmbd

# Проверка статуса
sudo systemctl status smbd

# Включение автозапуска
sudo systemctl enable smbd nmbd
```

### 2.4 Настройка брандмауэра (если есть)

```bash
# Разрешение портов Samba
sudo ufw allow 137/tcp
sudo ufw allow 137/udp
sudo ufw allow 138/tcp
sudo ufw allow 138/udp
sudo ufw allow 139/tcp
sudo ufw allow 139/udp
sudo ufw allow 445/tcp
sudo ufw allow 445/udp
```

## ⚙️ ЧАСТЬ 4: Настройка VK API

### 4.1 Редактирование конфигурации

```bash
# Редактирование app.py
nano app.py

# Редактирование bot_worker.py
nano bot_worker.py
```

### 4.2 Замена токенов

**Найдите и замените в обоих файлах:**

```python
# Было:
VK_TOKEN = 'YOUR_VK_TOKEN_HERE'
VK_GROUP_ID = 'YOUR_GROUP_ID'

# Стало (ваши данные):
VK_TOKEN = 'vk1.a.ABC123xyz...'      # Ваш токен VK
VK_GROUP_ID = '-123456789'            # ID группы с минусом!
```

### 4.3 Получение токена VK

1. Зайдите на [https://vk.com/dev](https://vk.com/dev)
2. Создайте Standalone-приложение
3. Получите Access Token с правами: `messages`, `offline`, `groups`
4. ID группы: из ссылки `https://vk.com/club123456789` → ID = `-123456789`
5. В настройках группы включите **«Сообщения сообщества»**

## 🔧 ЧАСТЬ 5: Настройка автозапуска (systemd)

### 5.1 Сервис Flask

```bash
# Создание сервиса
sudo nano /etc/systemd/system/laser-workshop.service
```

**Содержимое:**

```ini
[Unit]
Description=Laser Workshop CRM Phase 3
After=network.target

[Service]
Type=simple
User=andor
WorkingDirectory=/home/andor/laser-workshop-phase3
Environment="PATH=/home/andor/laser-workshop-phase3/venv/bin"
ExecStart=/home/andor/laser-workshop-phase3/venv/bin/python3 app.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 5.2 Сервис VK Бота

```bash
# Создание сервиса
sudo nano /etc/systemd/system/laser-bot.service
```

**Содержимое:**

```ini
[Unit]
Description=Laser Workshop VK Bot Phase 3
After=network.target laser-workshop.service

[Service]
Type=simple
User=andor
WorkingDirectory=/home/andor/laser-workshop-phase3
Environment="PATH=/home/andor/laser-workshop-phase3/venv/bin"
ExecStart=/home/andor/laser-workshop-phase3/venv/bin/python3 bot_worker.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

### 5.3 Включение сервисов

```bash
# Перезагрузка конфигурации
sudo systemctl daemon-reload

# Включение автозапуска
sudo systemctl enable laser-workshop.service
sudo systemctl enable laser-bot.service

# Запуск сервисов
sudo systemctl start laser-workshop.service
sudo systemctl start laser-bot.service

# Проверка статуса
sudo systemctl status laser-workshop.service
sudo systemctl status laser-bot.service
```

## 🌐 ЧАСТЬ 6: Подключение с ПК через Samba

### 6.1 Узнать IP адрес Raspberry Pi

```bash
# На Raspberry Pi
hostname -I
# Пример: 192.168.1.100
```

### 6.2 Подключение из Windows

**Способ 1: Через Проводник**

1. Откройте Проводник (`Win+E`)
2. В адресной строке введите: `\\192.168.1.100\`
3. Введите учётные данные:
   - Логин: `andor`
   - Пароль: `sambapass123`
4. Вы увидите папку `laser-workshop-phase3`

**Способ 2: Подключить как сетевой диск**

1. Проводник → Этот компьютер → **"Подключить сетевой диск"**
2. Буква диска: `Z:`
3. Папка: `\\192.168.1.100\laser-workshop`
4. ✅ **"Восстанавливать при входе в систему"**
5. Введите учётные данные при запросе

**Способ 3: Командная строка**

```cmd
net use Z: \\192.168.1.100\laser-workshop /user:andor sambapass123 /persistent:yes
```

### 6.3 Редактирование кода с ПК

| Редактор | Как использовать |
|----------|-----------------|
| **VS Code** | Файл → Открыть папку → `Z:\` |
| **Notepad++** | Файл → Открыть → `Z:\app.py` |
| **PyCharm** | File → Open → `Z:\` |

> ⚠️ **Важно:** После редактирования `.py` файлов перезапустите сервисы:
> ```bash
> sudo systemctl restart laser-workshop.service laser-bot.service
> ```

## ⚠️ ЧАСТЬ 9: Решение проблем

| Проблема | Решение |
|----------|---------|
| **Samba не видит папку** | Проверьте `sudo systemctl status smbd` |
| **Отказано в доступе** | Проверьте права: `sudo chown -R andor:andor ~/laser-workshop-phase3` |
| **Port 5000 занят** | Измените в `app.py`: `app.run(port=5001)` |
| **VK бот не работает** | Проверьте токен и права доступа в группе |
| **БД не создаётся** | Проверьте права на запись: `chmod -R 755 ~/laser-workshop-phase3` |
| **Нет доступа из сети** | Проверьте брандмауэр и IP: `hostname -I` |

### Диагностика БД

```bash
# Проверка таблиц
sqlite3 workshop.db ".tables"

# Проверка пользователей
sqlite3 workshop.db "SELECT username, role FROM users;"
```

## 📞 Поддержка

При возникновении проблем:

1. **Проверьте логи сервисов**
   ```bash
   sudo journalctl -u laser-workshop.service -n 100
   ```

2. **Убедитесь что VK токен действителен**
   - Права: `messages`, `groups`, `offline`
   - Сообщения сообщества включены

3. **Проверьте права доступа к папкам**
   ```bash
   ls -la ~/laser-workshop-phase3/
   chmod -R 755 ~/laser-workshop-phase3/
   ```

4. **Перезапустите сервисы**
   ```bash
   sudo systemctl restart laser-workshop.service laser-bot.service smbd nmbd
   ```

5. **Восстановите БД из бэкапа**
   ```bash
   cp backups/workshop_backup_YYYYMMDD.db workshop.db
   ```
